# Who is this for?

## Context

Audience shapes decisions throughout: whether the plugin needs onboarding UX, whether errors surface as friendly messages or dev-facing logs, whether the settings UI needs to be polished, how much compatibility range to target, and whether it ever gets published to AMO (addons.mozilla.org).

## Options (if any)

- **Personal use only**: No need to publish; rough edges are fine; can use developer mode install. Simpler distribution and no AMO review constraints.
- **Public / open source release**: Needs AMO publication at some point, proper onboarding, user-facing errors, permission explanations, potentially stricter review of native host integration.
- **Small circle (friends/community)**: Somewhere in between — shared as a GitHub download, not formally on AMO.

## Answer

Public release — both plugins intended for publication on addons.mozilla.org.

## Rationale

User stated "I plan on a public release."
